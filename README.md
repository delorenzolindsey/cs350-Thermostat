# cs350-Thermostat
CS350 final project 

## Summarize the project and what problem it was solving.

This project is a Thermostat that connects to an embedded system built from the class CS350 with SNHU. Within this system hardware there is a Raspberry Pi, solderless breadboard, Wires, 3 buttons, a temperature sensor, and a 16x2 LCD screen.

## What did I do well and how could I improve?

I believe I gathered a well understanding of the physical items `within my hardware`. Within the guides given it was required to have three buttons. The button location however, was blocked and required moving to a new location. This was easily fixed with the understanding of how to correctly set up each connecting wires and resistor. I could improve with `my troobleshooting`. My troobleshooting could process more smoothly if I was not working to quickly. The answers are often right in front of my with the simplest mistake.

## What tools did I add to support my work?

Sometimes the simplest tools can help. I have a simple `Notepad` with my commands for my Terminal. This helped me quickly copy and paste my commands to the terminal. Testing can become a lengthy process at times.
Also, sometimes simply `speaking up` can help drasticly. At first I had to ask for guidance but after a bit I understood and no longer need help. My strongest weakness I speaking up for help, but I pushed past this to be sure that my work can successfully become completed.

## What skills from this project will be transferable to other projects or work?

This was my first course that taught me `how to work with hardware`. All my previous classes focused on the software side. I believe this gave me I different appraoch to how machines truly run.

## How did I make this project maintainable, readable, and adaptable?

The base of the code was given to me. To be forward with this fact I left the the words `"TODO"` in the code can changed it to `"TODO COMPLETE"`. Its is forward with where my edits are and easy to spot for grading or correction. Comments are very important as they are a form of communication to any future developers reading the code later.
